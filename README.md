# Лабораторная работа 11. CI/CD и автоматизированное тестирование

## Выбранный вариант

Выбран вариант с GitHub Actions.

## Веб-страница

Учебная страница находится в `lab 11/public/index.html`. Это отдельная форма регистрации на курс с полями:

- имя;
- email;
- кнопка отправки;
- сообщение об ошибке или успешной отправке.

Production-код основного приложения не изменялся.

## UI-автотесты

Тесты находятся в `lab 11/tests/form-ui.test.js` и используют Selenium WebDriver.

Проверки:

- страница формы открывается и содержит ожидаемые элементы;
- отправка пустой формы показывает ошибку;
- невалидный email показывает ошибку;
- валидная форма показывает сообщение об успехе.

Запуск:

```bash
cd "lab 11"
npm test
```

В выводе тестов есть русские диагностические сообщения `Отправили` и `Получили`: они показывают действие в браузере, введенные данные и сообщение, которое вернула форма.

## GitHub Actions

Workflow находится в `.github/workflows/ci.yml`.

Он выполняет:

- запуск API-тестов лабораторной 10;
- запуск Selenium UI-тестов лабораторной 11;
- деплой `lab 11/public` на GitHub Pages только после успешных тестов в ветке `main`.

## Организация веток

Для выполнения сценария из задания:

```bash
git checkout -b dev
git push -u origin dev

git checkout -b fix dev
# внести изменение в учебную форму или тест
git add .
git commit -m "Update lab 11 form"
git push -u origin fix
```

Далее на GitHub:

1. Создать pull request `fix -> dev`.
2. Дождаться успешного прохождения GitHub Actions.
3. Выполнить merge в `dev`.
4. Создать pull request `dev -> main`.
5. Дождаться финального прохождения GitHub Actions.
6. Выполнить merge в `main`.
7. После push/merge в `main` workflow опубликует страницу на GitHub Pages.

## Проверка падения тестов

Для проверки пункта задания про ошибку можно временно изменить текст кнопки в `lab 11/public/index.html`. Тест `form page renders the expected controls` должен завершиться неуспешно, потому что ожидает текст `Отправить заявку`.
